## Naming Conventions

| Type | Convention | Example |
|---|---|---|
| Vue components | PascalCase | `TaskCard.vue` |
| Composables | camelCase with `use` prefix | `useIpc.js` |
| Stores | camelCase with `Store` suffix | `projectsStore.js` |
| Repositories | camelCase with `Repository` suffix | `tasksRepository.js` |
| IPC handlers | camelCase with `Handlers` suffix | `tasksHandlers.js` |
| IPC channel names | `domain:action` | `tasks:create` |
| CSS variables | `--zntt-{token}` | `--zntt-bg-primary` |

## Process Separation

### Renderer Process (`src/`)
- Vue 3 + Quasar components
- Pinia stores
- Router
- Never accesses Node.js APIs directly
- Communicates with main process only through `window.electronAPI`

### Main Process (`src-electron/`)
- Node.js environment
- SQLite access via repositories
- File system operations
- IPC handlers
- Window management
- Never imports Vue or Quasar

### Preload Script (`electron-preload.js`)
- The only bridge between renderer and main
- Exposes a whitelist of safe functions via `contextBridge`
- No direct Node.js exposure to the renderer
